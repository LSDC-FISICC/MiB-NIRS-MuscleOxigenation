System initialization and main control loop Initializes all peripherals in sequence:\+
\begin{DoxyEnumerate}
\item {\bfseries{Clock}}:\+ PLL to 64 MHz (HSI 8 MHz × 16)
\item {\bfseries{GPIO}}:\+ Status LED on PB3 (push-\/pull output)
\item {\bfseries{I2\+C1}}:\+ 400 k\+Hz fast-\/mode on PB6 (SCL), PB7 (SDA)
\item {\bfseries{Sensor}}:\+ MAX30101 Sp\+O2 Lite mode — Red + IR at 50 Hz, 1.\+6 mA each
\item {\bfseries{UART}}:\+ USART2 at 460800 baud (PA2=TX, PA15=RX)
\item {\bfseries{Timer}}:\+ Sys\+Tick at 50 Hz (20 ms period), enabling the acquisition ISR
\end{DoxyEnumerate}

After initialization, the main loop waits for data\+\_\+ready (set by Sys\+Tick ISR) and transmits each Red/\+\+IR sample pair over UART as a CSV string. All sensor acquisition runs in the ISR; main only handles transmission.

\begin{DoxyNote}{Note}
Initialization order is critical:\+ I2C must be configured before MAX30101, and UART before Sys\+Tick to avoid transmitting before the port is ready. 
\end{DoxyNote}
\begin{DoxyWarning}{Warning}
Enabling Sys\+Tick (last step) immediately arms the ISR. Any initialization that must complete before the first ISR fires should precede Sys\+Tick\+\_\+\+Config(). 
\end{DoxyWarning}
